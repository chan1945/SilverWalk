# 위험도 0 포인트의 잠재 위험도 예측 전략

## 목적

이 프로젝트의 핵심 목표는 이미 사고가 발생한 포인트를 다시 맞히는 것이 아니라, 원본 데이터에서 `위험도 = 0`인 포인트 중에서도 주변 환경 특성상 잠재적으로 위험한 포인트를 찾아내는 것이다.

따라서 지도에서 최종적으로 보고 싶은 대상은 아래와 같다.

```text
원본 위험도 = 0
AND 최종 위험도 점수 백분율 > 10%
```

즉, 사고 이력은 없지만 모델이 상대적으로 위험하다고 판단한 포인트를 시각화해야 한다.

## 현재 방식의 한계

현재 원본 데이터는 `위험도 = 0`인 행이 매우 많다. 이런 데이터에서 단순 회귀 모델을 학습하면 모델이 대부분의 포인트를 0 또는 0에 가까운 값으로 예측해도 전체 평균 성능이 좋아 보일 수 있다.

특히 `MAE`, `RMSE`, `R2` 같은 전체 회귀 지표는 데이터 대부분을 차지하는 0 위험도 포인트의 영향을 크게 받는다. 그 결과 모델이 실제로 위험한 포인트를 잘 찾지 못해도 평균 지표는 양호하게 나올 수 있다.

또한 현재 `위험도`는 진짜 위험도의 완전한 정답이라기보다 관측 기간 동안 발생한 사고 이력에 가깝다.

```text
위험도 = 0
```

은 “안전하다”가 아니라 “관측 기간 동안 사고가 기록되지 않았다”로 해석해야 한다.

## 쉬운 설명

현재 문제는 시험 문제로 비유하면 쉽다.

전체 100문제 중 95문제의 정답이 `0`이라고 가정한다. 이때 어떤 모델이 모든 문제에 `0`이라고 답하면 95문제를 맞힌다. 겉으로 보면 성능이 95%라서 좋아 보인다.

하지만 우리가 원하는 모델은 이런 모델이 아니다.

```text
모든 곳을 안전하다고 찍는 모델
```

이 아니라 아래 모델이 필요하다.

```text
사고가 아직 안 난 곳 중에서도,
사고가 난 곳과 환경이 비슷해서 위험해 보이는 곳을 찾아내는 모델
```

따라서 해결 방향은 두 가지를 바꾸는 것이다.

첫째, 모델 성능을 전체 평균 오차로만 보지 않는다.

전체 데이터 대부분이 `위험도 = 0`이면 전체 평균 오차는 모델이 0만 찍어도 좋아 보인다. 그래서 아래 질문으로 평가해야 한다.

```text
모델이 위험하다고 상위권에 올린 포인트들이 실제 사고 위험 패턴과 관련이 있는가?
```

즉, `MAE`, `RMSE`만 보는 것이 아니라 `최종위험도점수_percent` 상위 포인트, Top-K, 고위험 포인트 재현율, 공간 검증 등을 같이 봐야 한다.

둘째, 지도에 표시할 때 이미 사고가 난 곳은 제외한다.

학습은 사고가 난 곳의 패턴을 배우기 위해 `위험도 > 0` 데이터를 참고해야 한다. 하지만 최종 지도에서는 이미 사고가 난 곳을 다시 보여주는 것이 목적이 아니다. 따라서 예측 후 아래 조건으로 필터링한다.

```text
원본 위험도 = 0
AND 최종 위험도 점수 백분율 > 10%
```

이렇게 하면 지도에는 아래 포인트만 남는다.

```text
과거 사고 이력은 없지만,
모델이 환경 특성상 위험하다고 본 포인트
```

정리하면 문제를 푸는 핵심은 “0을 잘 맞히는지”를 성능으로 보지 않는 것이다. 대신 “모델이 위험하다고 고른 상위 포인트가 의미 있는지”를 평가하고, 최종 시각화에서는 `위험도 = 0`인 포인트만 남기는 방식으로 목적에 맞춘다.

## 목표 재정의

모델이 풀어야 하는 문제는 다음과 같이 정의하는 것이 적절하다.

```text
사고 이력이 없는 포인트 중에서,
주변 환경 특성이 사고 발생 포인트와 유사하거나 위험 요인이 높은 포인트를 찾는다.
```

이때 `POINT_ID`, `위도`, `경도`는 모델 입력으로 사용하면 안 된다. 이 값들은 위치 표시와 결과 병합을 위한 키이며, 모델이 특정 좌표나 ID를 암기하게 만들 수 있다.

## 권장 데이터 흐름

아래 흐름을 기준으로 구현한다.

```text
1. 원본 데이터 로드
2. POINT_ID, 위도, 경도, 위험도는 모델 입력에서 제외
3. 나머지 환경 feature만 전처리
4. 원본 위험도 = 0인 포인트만 학습된 MLP로 예측
5. 모델 1에서 `사고발생확률_p` 예측
6. 모델 2에서 `조건부위험도_r` 예측
7. `최종위험도점수 = 사고발생확률_p * 조건부위험도_r` 계산
8. 위험도 = 0 예측 대상 안에서 `최종위험도점수_percent` 생성
9. 원본 위험도 = 0 조건을 결과 CSV에 위험도_actual로 보존
10. 최종위험도점수_percent > 10인 포인트만 지도에 시각화
```

`최종위험도점수_percent`는 두 모델 결합 점수인 `최종위험도점수`를 백분율로 환산한 값이다.

```text
최종위험도점수 = 사고발생확률_p * 조건부위험도_r
최종위험도점수_percent = 최종위험도점수 / max(전체 최종위험도점수) * 100
```

이 값은 확률이 아니라 전체 예측 포인트 안에서의 상대 위험도 백분율이다.

## 예측 결과 CSV 권장 컬럼

전체 예측 결과 파일은 아래 컬럼을 포함하는 것이 좋다.

| 컬럼 | 용도 | 모델 입력 여부 |
|---|---|---|
| `POINT_ID` | 원본 포인트 식별자, 지도/병합용 | 제외 |
| `위도` | 지도 표시용 | 제외 |
| `경도` | 지도 표시용 | 제외 |
| `위험도_actual` | 원본 위험도. 사고 이력 0 필터링 및 평가용 | 제외 |
| `사고발생확률_p` | 모델 1이 예측한 사고 발생 확률 점수 | 결과 |
| `조건부위험도_r` | 모델 2가 예측한 사고 발생 시 위험도 크기 | 결과 |
| `최종위험도점수` | `사고발생확률_p * 조건부위험도_r` | 결과 |
| `최종위험도점수_percent` | 전체 결합 점수 기준 상대 위험도 백분율 | 결과 |

중요한 점은 `위험도_actual`을 모델 입력에 넣지 않는 것이다. 이 컬럼은 예측 후 필터링과 평가에만 사용한다.

## 지도 시각화 기준

지도에는 아래 조건을 만족하는 포인트만 표시한다.

```text
위험도_actual = 0
AND 최종위험도점수_percent > 10
```

이 기준을 쓰면 이미 사고가 발생했던 포인트가 아니라, 사고 이력은 없지만 모델이 위험하다고 판단한 포인트를 볼 수 있다.

현재 예측 결과에서는 `50% 초과` 기준이 매우 엄격해서 표시 포인트가 7개 수준으로 줄어든다. 따라서 지도 표시 시작 기준은 `10% 초과`로 낮추고, 등급은 아래처럼 해석한다.

| 구간 | 등급 | 의미 |
|---|---|---|
| `10% 초과 ~ 30% 미만` | 주의 | 지도에 표시해 검토할 잠재 위험 후보 |
| `30% 이상 ~ 50% 미만` | 경고 | 상대 위험도가 높은 후보 |
| `50% 이상` | 위험 | 최상위 위험 후보 |

툴팁에는 최소한 아래 정보를 표시한다.

| 항목 | 설명 |
|---|---|
| `POINT_ID` | 포인트 식별자 |
| `위험도_actual` | 원본 사고 이력 위험도. 지도 표시 대상은 0이어야 함 |
| `사고발생확률_p` | 모델 1의 사고 발생 확률 점수 |
| `조건부위험도_r` | 모델 2의 사고 발생 시 위험도 크기 |
| `최종위험도점수` | 두 모델 결합 점수 |
| `최종위험도점수_percent` | 상대 위험도 백분율 |

## 평가 방법

이 문제에서는 전체 MAE/RMSE만으로 모델을 평가하면 안 된다. 다음 평가를 추가해야 한다.

| 평가 항목 | 설명 |
|---|---|
| Zero subset ranking | `위험도_actual = 0`인 포인트 중 예측 상위 지점을 별도로 분석 |
| Top-K review | `최종위험도점수_percent` 상위 100개, 500개, 1000개 포인트의 공간 분포 검토 |
| 고위험 재현율 | 실제 `위험도 > 50` 포인트가 예측 상위 구간에 얼마나 포함되는지 확인 |
| Precision@K | 예측 상위 K개 중 실제 사고 이력이 있는 포인트 비율 |
| 공간 검증 | 가까운 포인트가 train/test에 동시에 섞여 생기는 위치 누수를 줄이기 위해 구 단위 또는 공간 블록 단위 검증 |
| 시간 검증 | 사고 데이터에 연도/기간 정보가 있으면 과거로 학습하고 이후 기간 사고를 맞히는지 평가 |

특히 가장 이상적인 검증은 시간 분리 검증이다.

```text
과거 기간 데이터로 학습
이후 기간에 사고가 새로 발생한 포인트를 예측 상위권에서 찾는지 평가
```

이렇게 해야 “사고 이력이 없던 지역의 미래 위험도”를 더 직접적으로 검증할 수 있다.

## 모델링 개선 방향

현재 단일 MLP 회귀 모델은 시작점으로 사용할 수 있지만, zero-heavy 데이터에는 다음 방식도 고려해야 한다.

| 방식 | 설명 | 장점 | 주의점 |
|---|---|---|---|
| Sample weight | `위험도 > 0` 또는 고위험 행에 더 큰 학습 가중치 부여 | 0만 예측하는 경향 완화 | 가중치가 과하면 고위험 과대예측 가능 |
| 2-stage model | 1단계 사고 발생 가능성 분류, 2단계 발생 시 위험도 회귀 | zero-heavy 문제에 적합 | 모델과 평가가 복잡해짐 |
| Ranking objective | 절대 위험도보다 위험 순위를 잘 맞히도록 학습 | 지도 상위 위험지점 선정에 적합 | 구현 난이도 증가 |
| 고위험 분류 모델 | `위험도 > 50` 또는 상위 분위수를 분류 | 위험지역 탐지 목적에 직접적 | threshold 선택 근거 필요 |

현재 목표에는 단순 평균 오차 최소화보다 위험 순위 품질이 더 중요하다. 따라서 예측값의 절대 크기보다 `최종위험도점수_percent` 상위 포인트가 실제로 위험 요인을 많이 가진 지점인지 검토해야 한다.

## 구현 우선순위

### 1단계: 현재 파이프라인 보정

- 전체 예측 결과에 원본 `위험도`를 `위험도_actual`로 함께 저장한다.
- 모델 입력에는 `위험도_actual`, `POINT_ID`, `위도`, `경도`를 사용하지 않는다.
- Streamlit 지도는 `위험도_actual = 0 AND 최종위험도점수_percent > 10`만 표시한다.

### 2단계: 평가 지표 추가

- `위험도_actual = 0` 포인트의 예측 상위 목록을 별도로 저장한다.
- 실제 사고 이력 포인트가 예측 상위권에 얼마나 들어오는지 `Precision@K`, `Recall@K`로 본다.
- 구별, 행정동별, 공간 블록별로 예측 상위 포인트가 과도하게 몰리는지 확인한다.

### 3단계: 검증 방식 개선

- 가능하면 연도별 사고 데이터를 확보해 시간 분리 검증을 수행한다.
- 시간 정보가 없으면 구 단위 또는 공간 격자 단위로 train/test를 나누어 위치 누수를 줄인다.

### 4단계: 모델 개선

- sample weight를 적용한 MLP 회귀 모델을 실험한다.
- 사고 발생 가능성 분류 모델과 위험도 회귀 모델을 나누는 2-stage 구조를 실험한다.
- 최종 지도용 점수는 확률 또는 상대 위험도 백분율로 통일한다.

## 결론

현재 목표에서는 “위험도 0을 잘 맞히는 모델”이 아니라 “위험도 0으로 관측된 포인트 중 잠재적으로 위험한 포인트를 잘 골라내는 모델”이 필요하다.

따라서 최종 시각화 기준은 다음으로 두는 것이 적절하다.

```text
위험도_actual = 0
AND 최종위험도점수_percent > 10
```

이 방식은 이미 사고가 발생한 지점을 다시 보여주는 것이 아니라, 사고 이력이 없지만 환경적으로 위험할 수 있는 후보 지점을 지도에 표시하는 목적에 더 맞다.
