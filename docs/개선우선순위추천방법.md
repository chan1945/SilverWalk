
1. 위험도_actual = 0 AND 최종위험도점수_percent > 10 포인트 선택
2. 각 포인트에 대해 SHAP 값 계산
3. 위험도를 올린 feature만 추출
4. feature를 실제 개선 항목으로 매핑
5. SHAP 기여도가 큰 순서대로 개선 우선순위 추천

개선 추천 컬럼들 :
- 과속방지턱개수
- 횡단보도개수
- 신호등설치여부
- 노인장애인보호구역여부
- 가로등개수
- 보행자우선도로여부
- CCTV개수
- 제한속도
- 횡단보도예고표시여부

전체 feature 기준으로 SHAP 적용 후 개선 추천 컬럼만 필터링 

두 모델에서 공통으로 위험 기여가 큰 feature 를 추천하며

내림차순으로 정렬하여 개선우선순위 3개를 추천

예시
- 1. 과속방지턱 추가 
- 2. 횡단보도 추가
- 3. 노인/장애인 보호구역 지정

각 컬럼에 대한 개선우선순위 내용은 다음과 같이 매핑
| feature | 개선내용 |
| --- | --- |
| 과속방지턱개수 | 과속방지턱 추가 |
| 횡단보도개수 | 횡단보도 추가 |
| 신호등설치여부 | 신호등 설치 |
| 노인장애인보호구역여부 | 노인/장애인 보호구역 지정 | 
| 가로등개수 | 가로등 추가 |
| 보행자우선도로여부 | 보행자우선도로 지정 |
| CCTV개수 | CCTV 추가 |
| 제한속도 | 제한속도 하향 |
| 횡단보도예고표시여부 | 횡단보도 예고표시 설치 |

## 구현 산출물

개선우선순위 추천은 아래 스크립트로 생성한다.

```bash
python scripts/explain/recommend_improvements.py
```

결과 CSV는 아래 경로에 저장한다.

```text
artifacts/recommendations/point_improvement_recommendations.csv
```

지도는 이 파일이 있으면 포인트 툴팁에 `개선우선순위1`, `개선우선순위2`, `개선우선순위3`을 함께 표시한다.
