# original_train_data MLP 전처리 계획

## 데이터 요약

- 대상 파일: `data/original_train_data/seoul_road_points.csv`
- 데이터 크기: 450,663행, 106열
- 결측치: 없음
- 중복 `POINT_ID`: 없음
- 전체 컬럼 타입: `int64` 103개, `float64` 3개
- 타겟 컬럼: `위험도`
- `위험도` 분포: 0값 360,787개, 0 초과 89,876개, 50 초과 701개
- `위험도` 분위수: p50 0.000, p75 0.000, p90 1.168, p95 2.472, p99 12.700, max 377.872

`위험도`는 0이 많은 긴 꼬리 분포이므로 기본 학습은 회귀 문제로 두되, 타겟은 `log1p(위험도)`로 변환한다. 예측 결과를 화면에 표시할 때는 `expm1(pred)`로 원래 위험도 스케일을 복원한다.

## 공통 전처리 원칙

1. Train/validation/test split을 먼저 수행한다.
2. `StandardScaler`, 분위수 클리핑 기준, 결측 대체값은 train split에서만 fit한다.
3. `POINT_ID`, `위도`, `경도`는 모델 입력에서 제외하고, 예측 결과 병합 및 지도 표시용 키로만 보존한다.
4. `위험도`는 정답 타겟이므로 입력 피처에서 반드시 제외한다.
5. 카운트 피처는 대부분 0이 많고 오른쪽 꼬리가 길기 때문에 `log1p` 변환 후 `StandardScaler`를 적용한다.
6. 0/1 여부 컬럼은 0/1 정수 그대로 사용한다.
7. 추론 데이터에서 결측이 발생하면 이진 플래그는 0 또는 train 최빈값, 연속/카운트 피처는 train 중앙값으로 대체한다.

| `신호등설치여부` | 이진 플래그 | 입력 피처 | 0/1 정수로 유지, 결측 발생 시 0 또는 최빈값 대체 | 이미 MLP에 적합한 범위의 이진 변수. |
