# 기법별 성능평가

## 목적

이 문서는 SilverWalk 프로젝트에서 여러 학습 기법을 적용하고, 모델 성능을 같은 기준으로 비교하기 위한 평가 계획을 정리한다.

현재 프로젝트는 모델을 2개로 나누어 사용한다.

```text
모델 1: 사고 발생 가능성 분류
모델 2: 사고 발생 시 위험도 크기 회귀
최종 점수: 사고발생확률_p * 조건부위험도_r
```

따라서 성능평가도 모델 1, 모델 2, 최종 2-stage 결과를 나누어 진행해야 한다.

## 평가 기본 원칙

기법별 성능을 비교할 때는 아래 원칙을 지킨다.

1. 같은 train/validation/test split을 사용한다.
2. 전처리 방식은 동일하게 유지한다.
3. 모델 입력에서 `POINT_ID`, `위도`, `경도`, `위험도`는 제외한다.
4. test set은 최종 성능 평가에만 사용한다.
5. accuracy나 RMSE 하나만 보고 판단하지 않는다.
6. 최종 목적에 맞게 Top-K 계열 지표를 함께 본다.
7. 성능뿐 아니라 SHAP 개선우선순위 추천의 안정성도 함께 확인한다.

현재 split 방식은 K-fold가 아니라 고정 train/validation/test 분할이다.

```text
train: 70%
validation: 15%
test: 15%
```

## 현재 기준 모델

현재 기준 모델은 아래 2-stage 구조이다.

| 구분 | 모델 | 학습 대상 | 출력 |
|---|---|---|---|
| 모델 1 | MLP 분류 모델 | 전체 데이터 | `사고발생확률_p` |
| 모델 2 | MLP 회귀 모델 | `위험도 > 0` 데이터 | `조건부위험도_r` |
| 최종 | 2-stage 결합 | `위험도_actual = 0` 포인트 | `최종위험도점수 = p * r` |

현재 기준 성능은 아래 수준이다.

| 모델 | 주요 지표 | 현재 성능 |
|---|---|---|
| 모델 1 | ROC-AUC | 약 `0.91` |
| 모델 1 | PR-AUC | 약 `0.74` |
| 모델 1 | F1-score | 약 `0.68` |
| 모델 1 | Precision@1000 | 약 `0.95` |
| 모델 2 | log-scale R2 | 약 `0.60` |
| 모델 2 | risk-scale R2 | 약 `0.54` |
| 모델 2 | 고위험 Top300 Recall | 약 `0.82` |

이 값을 baseline으로 두고 다른 기법을 비교한다.

## 모델 1 평가 지표

모델 1은 `사고발생 = 위험도 > 0`을 예측하는 분류 모델이다.

### 주요 지표

| 지표 | 설명 | 판단 기준 |
|---|---|---|
| PR-AUC | 불균형 데이터에서 사고 발생 포인트를 얼마나 잘 찾는지 | 높을수록 좋음 |
| ROC-AUC | 사고 발생/미발생을 전반적으로 구분하는 능력 | 높을수록 좋음 |
| Precision | 사고 발생으로 예측한 포인트 중 실제 사고 발생 비율 | 높을수록 좋음 |
| Recall | 실제 사고 발생 포인트 중 모델이 찾은 비율 | 높을수록 좋음 |
| F1-score | Precision과 Recall의 균형 | 높을수록 좋음 |
| Accuracy | 전체 정답률 | 참고 지표 |
| Precision@K | 예측 상위 K개 중 실제 사고 발생 포인트 비율 | 높을수록 좋음 |
| Recall@K | 실제 사고 발생 포인트 중 예측 상위 K개에 포함된 비율 | 높을수록 좋음 |

### 모델 1에서 가장 중요한 지표

모델 1에서는 아래 지표를 우선적으로 본다.

```text
1. PR-AUC
2. Precision@K
3. Recall@K
4. F1-score
5. Precision / Recall
6. Accuracy
```

Accuracy는 보조 지표로만 사용한다. `위험도 = 0` 데이터가 많기 때문에, 모델이 사고 미발생을 많이 맞히면 accuracy가 높아 보일 수 있다.

## 모델 2 평가 지표

모델 2는 `위험도 > 0` 데이터만 사용해 사고 발생 시 위험도 크기를 예측하는 회귀 모델이다.

### 주요 지표

| 지표 | 설명 | 판단 기준 |
|---|---|---|
| log-scale MAE | `log1p(위험도)` 기준 평균 절대 오차 | 낮을수록 좋음 |
| log-scale RMSE | `log1p(위험도)` 기준 큰 오차 반영 | 낮을수록 좋음 |
| log-scale R2 | 로그 스케일 설명력 | 높을수록 좋음 |
| risk-scale MAE | 원래 위험도 스케일 평균 절대 오차 | 낮을수록 좋음 |
| risk-scale RMSE | 원래 위험도 스케일 큰 오차 반영 | 낮을수록 좋음 |
| risk-scale R2 | 원래 위험도 스케일 설명력 | 높을수록 좋음 |
| high-risk Precision@K | 예측 위험도 상위 K개 중 실제 고위험 포인트 비율 | 높을수록 좋음 |
| high-risk Recall@K | 실제 고위험 포인트가 예측 상위 K개에 들어온 비율 | 높을수록 좋음 |

### 고위험 기준

모델 2에서는 실제 고위험 포인트를 아래처럼 정의한다.

```text
actual_high = 실제 위험도 > 50
```

모델 2는 회귀 모델이므로 F1-score는 사용하지 않는다. 고위험 구간 평가는 `위험도 > 50` 포인트가 예측 상위 K개 안에 얼마나 포함되는지로 판단한다.

### 모델 2에서 가장 중요한 지표

```text
1. high-risk Recall@K
2. high-risk Precision@K
3. log-scale MAE / RMSE
4. risk-scale MAE / RMSE
5. R2
```

## 최종 2-stage 평가 지표

최종 점수는 아래와 같이 계산한다.

```text
최종위험도점수 = 사고발생확률_p * 조건부위험도_r
```

최종 결과는 `위험도_actual = 0`인 포인트 중 잠재 위험 후보를 찾는 데 사용한다.

### 주요 지표

| 지표 | 설명 |
|---|---|
| 최종점수 Top-K 검토 | 최종위험도점수 상위 포인트가 의미 있는 공간 분포를 가지는지 확인 |
| 최종점수 Precision@K | 사고 이력 포인트를 평가 대상으로 삼았을 때 상위 K개 적중률 |
| 최종점수 Recall@K | 실제 고위험 포인트가 상위 K개에 얼마나 포함되는지 |
| 표시 후보 수 | `위험도_actual = 0 AND 최종위험도점수_percent > 10` 포인트 수 |
| 구별 분포 | 특정 구에 과도하게 몰리는지 확인 |
| SHAP 추천 안정성 | 같은 유형의 포인트에서 추천 우선순위가 과도하게 흔들리지 않는지 확인 |

최종 2-stage 결과는 단순히 test set 점수만 보는 것이 아니라 지도에 표시되는 후보가 실제로 검토 가능한 형태인지 함께 봐야 한다.

### 모델 1 undersampling 실험 기준

현재 데이터는 `위험도 = 0`이 많기 때문에 모델 1에 undersampling을 실험할 수 있다.

추천 실험 순서는 아래와 같다.

```text
baseline
undersample 1:3
undersample 1:2
undersample 1:1
```

무조건 1:1 undersampling을 기본 적용하지 않는다. 1:1은 negative 정보를 많이 버리기 때문에 성능과 SHAP 안정성을 확인한 뒤 결정한다.

## 최종 2-stage 실험 기법

| 실험명 | 설명 | 기대 효과 | 주의점 |
|---|---|---|---|
| `two_stage_baseline` | 현재 `p * r` 방식 | 기준 결과 | 없음 |
| `two_stage_p_weighted` | `p^alpha * r` | 사고 발생 가능성 비중 조절 | alpha 선택 필요 |
| `two_stage_r_weighted` | `p * r^beta` | 위험도 크기 비중 조절 | beta 선택 필요 |
| `two_stage_percent_threshold_tuning` | 지도 표시 기준 10/20/30% 비교 | 표시 후보 수 조절 | 기준 설명 필요 |
| `two_stage_top_k_display` | percent 기준 대신 Top-K 표시 | 후보 수 고정 | 50/70/90 등급 해석 어려움 |

현재 지도 시각화는 `최종위험도점수_percent > 10` 기준을 사용한다. 이 기준은 후보 수와 해석 가능성을 함께 고려해 조정할 수 있다.

## 우선 실험 순서

처음부터 모든 기법을 적용하지 않고 아래 순서로 진행한다.

```text
1. 모델 1 baseline 재현
2. 모델 1 undersampling 1:3, 1:2, 1:1 비교
3. 모델 1 class weight 비교
4. 모델 2 baseline 재현
5. 모델 2 high-risk oversampling 비교
6. 최종 2-stage 점수와 지도 후보 비교
7. SHAP 개선우선순위 안정성 비교
```

우선순위가 높은 실험은 모델 1 undersampling이다. 현재 데이터에서 `위험도 = 0`이 많고, 모델 1이 사고 발생 가능성 `p`를 결정하므로 최종 점수에 직접적인 영향을 주기 때문이다.
